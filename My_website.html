Skills

  • Here is a list of the skills i have gained during the
    year I worked for Captain's eye company:


  • Linux - UBUNTU

    • Most of the servers needed to have UBUNTU on them so our system could work proporly,
    therefore - I have worked almost 90% of the day on UBUNTU or powershell in order to access
    the systems on the vessels and fix or update the program according to the vessel's needs.



  • Bash and Shell scripting

    • In order to make the installation department more usefull and quick,
    I have created automations in Bash that helped the team install the softwere more easily,
    and also documented the hardware in the Database. All the day-to-day work had to be in the powershell or UBUNTU and therefore involved a lot of shell scripting.



  • Database (MongoDB)

    • All the parameters of the vessels were documented and maintained through the Database,
    every vessel had its own page, and there was also a control page for all the vessles together,
    most of my work in the Database was to upload new CCTV cameras from the vessels into the Database,
    and to change parameters according to the needs of the vessel.



  • Amazon Web Services (AWS)

    • All the vessels had to be connected to the cloud in order to function proporly,
    I had to ensure that every vessel had its own credential in order to access the cloud,
    I uplouded all my automations to the cloud so it will be easier to download them no matter
    where the vessel is currently sailing.



  • VPN - Wiregaurd

    • In order to prevent unwanted visitors or malicious malware inside the company servers,
    every vessel had its own Wiregaurd IP that you could access only after connecting to the VPN.
    I had to ensure that every vessel got its unique and accessible IP.



  • Domain name system (DNS)

    • In order to make it easier to access every vessel,
    instead of writing the IP of the server every time, I connected between the IP
    of the server and the name of the vessel so it will be easier to access.



  • Cameras, Switches and Access points

    • The company's system included Cameras, Switches and Access points that were connected
    to our servers.
    I have configured them before we sent them to the vessel,
    and after they arrived to the vessel - I made sure that I was able to see
    them online and added the Cameras to the Database so they will appear on the Website.



  • Docker

    • In order to make sure that all the vessels have the latest updates of the softwere,
    we created containers for each and every different subject that the softwere involved.
    This way, it was easier to control the softwere on all the vessels together.
    I had to change docker compose and make sure the containers are up on the latest version.



  • Network configurations

    • All the network configurations were made or at least checked by me,
    both on shore and on the vessels themselves. I would access through the DNS or WG, or through teamviewer or RDP
    in a case of a problem.
